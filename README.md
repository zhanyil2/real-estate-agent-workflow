## 一眼看懂

这是一个专门为中国房地产经纪人设计的 `OpenClaw + 小红书 + Notion + QQ` 自动化工作流。

你录入房源，它负责把房源变成内容，再把内容变成持续获客动作。群聊里的房源信息它帮你盯，客户找房它帮你搜：

| 步骤 | 动作 |
|:---:|---|
| 1 | 新房源进入 Notion 房源数据库（手动录入或企微群聊自动抓取） |
| 2 | AI 自动生成适合小红书的文案与标签 |
| 3 | 自动准备封面图或调用已有房源图 |
| 4 | 通过企微/QQ 指令或定时任务（每日 17:00）触发发布 |
| 5 | 客户找房时自动匹配数据库，无结果则群聊广播求助 |
| 6 | 互动沉淀、线索记录，形成可复盘的数据闭环 |

不只是"小红书发帖工具"——它是从房源获取到客户成交的全链路助手。

**适合**：二手房经纪人 / 租赁经纪人 / 新房渠道顾问 / 个人 IP 型房产博主 / 社媒流量转私域的小团队

## 它为中国房产经纪人带来的价值

很多经纪人的时间，不是花在成交上，而是花在"重复搬运"上：

- 房源有了，但还没来得及写内容
- 内容写了，但没有合适封面
- 想发小红书，但要手动整理素材
- 发出去后，评论跟进不及时
- 客户意向散落在聊天、表格、脑子里

这个工作流的核心价值，就是把这些碎片动作收口成一个统一流程。

### 1. 更快把房源变成内容

新增房源后，系统可以自动读取房源信息，生成适合小红书风格的标题、正文、标签和封面图，大幅减少"从空白开始写"的时间。

### 2. 更稳定地保持发帖频率

经纪人最怕"忙成交时停更，停更后没线索"。这个流程支持：

- QQ 指令触发即时发帖
- 每天北京时间 `17:00` 自动检查新房源并发帖

这样就算当天很忙，也不会完全断更。

### 3. 更及时地承接社媒线索

评论、互动、潜在线索会进入结构化记录，不再只是零散存在聊天窗口里，方便后续筛选"谁只是看看、谁是真需求、谁值得马上跟进"。

### 4. 更广的房源渠道

企微群聊里每天有大量房源信息转瞬即逝。系统自动监听群消息，识别出房源信息后结构化提取并存入数据库，不再靠人肉翻聊天记录。

### 5. 客户找房秒级响应

客户说"帮我找望京两居500万以内"，系统先搜自有数据库，有就直接推荐；没有就自动在群聊里广播求助。从"我回头帮你问问"变成"我现在就帮你找"。

### 6. 更适合中国本地业务场景

它围绕国内经纪人的真实工作方式设计：

- 用 `企业微信` 做群聊监控和客户沟通
- 用 `小红书` 做内容获客
- 用 `QQ` 作为机器人触发入口
- 用 `Notion` 做内容库、房源库、线索库
- 用适合中文房产语境的内容风格输出

### 这套系统最适合提升的 3 件事

#### 1. 发帖效率

对很多经纪人来说，真正难的不是"会不会写"，而是"每天有没有时间稳定写、稳定发"。这套系统把房源信息直接转成可发布内容，让发帖从临时起意变成流程动作。

#### 2. 跟进速度

内容发出去之后，真正有价值的是互动后的承接。把线索记录下来，比单纯多发几篇更重要。系统把互动、线索、状态统一留在数据层里，更适合后续跟进。

#### 3. 内容标准化

同一个团队里，不同经纪人的内容质量常常不稳定。用统一的房源字段、统一的内容结构、统一的发布动作，可以明显减少"今天写得好、明天全靠发挥"的问题。

## 工作流一览

![工作流图](assets/xhs-agent-workflow.jpg)

### 当前链路

1. 把新房源录入 `房源列表 Property Listings`
2. 系统识别 `XHSPostStatus = new` 的房源
3. 自动生成适合小红书的房源推荐文案
4. 自动生成或调用现有封面图片
5. 发布到小红书
6. 把互动与线索继续沉淀到 Notion

## 企业微信群聊能力

这是本系统区别于纯"发帖工具"的核心能力。经纪人的日常工作围绕群聊展开，系统直接嵌入这个场景。

### 能力 1：群聊房源自动入库

群里有人发了一条房源信息：

```
望京西园四区，2室1厅，89平，精装修，南北通透，中楼层，580万
```

系统自动识别、提取结构化数据、查重、写入 Notion 房源数据库，并回复确认：

```
✅ 已录入房源
📍 望京西园四区 2室1厅 89㎡
💰 580万 | 来源: 群聊
```

不需要你手动复制粘贴，不需要你打开 Notion 填表。

### 能力 2：客户找房智能匹配

客户说"帮我找望京的两居室，500万以内"，系统会：

1. 先搜索 Notion 房源数据库
2. 有匹配 → 直接列出推荐
3. 无匹配 → 自动在群聊中发布求助消息

```
🔍 房源求助
📍 区域：望京  🏠 户型：2室  💰 预算：500万以内
如有合适房源请回复，谢谢🙏
```

从"我回头帮你问问"变成实时响应。

### 能力 3：小红书自动找群

系统每周自动在小红书搜索房产经纪人群聊邀请帖，发现后：

- 点赞 + 评论互动获取入群方式
- 将群线索记录到 Notion Social Leads 数据库
- 汇报发现结果

帮你持续扩展房源渠道。

## 对经纪人的实际使用场景

### 场景 1：当天新上 2 套优质房源

你只需要把房源录入到 Notion，系统就能在固定时间自动发帖，帮你保持内容输出频率。

### 场景 2：下班前想快速补一条内容

你可以直接给 QQ 机器人发一句：

```text
发房源
```

系统会读取新的房源并执行发帖流程。

### 场景 3：团队希望把内容获客做成标准动作

这套流程可以把"写文案的人""发帖的人""跟评论的人"之间的交接减少，让房源、内容、线索都留在统一系统里。

### 场景 4：群里刷了一屏房源信息

你加了十几个房源互通群，每天几百条消息，重要的房源信息一刷就过去了。现在系统帮你自动盯着，识别出的房源直接存入数据库，你空下来打开 Notion 就能看到今天所有新增房源。

### 场景 5：客户突然问"有没有望京的房子"

以前要翻群聊记录、翻表格、问同事。现在直接在企微里说"找望京两居500万"，系统秒级搜索数据库给结果。没有匹配的话，自动帮你在群里发求助。

### 场景 6：门店负责人想看结果而不是盯过程

负责人不需要每天追着问"今天发了没有""这个客户是谁接的"，而是直接通过结构化数据看到：

- 哪些房源正在排队待发
- 哪些内容已经发布
- 哪些互动正在转化成客户
- 哪些线索应该优先跟进

## 快速使用

### 1. 新增房源

在 `房源列表 Property Listings` 数据库中新增一条记录，并填写：

- `Name`
- `PropertyType`
- `District`
- `Area`
- `Price`
- `PriceUnit`
- `Rooms`
- `Size`
- `Highlights`
- `ImagePaths`
- `XHSPostStatus = new`

### 2. 选择发帖方式

方式 A：QQ 中发送

```text
发房源
```

方式 B：等待系统在每天 `17:00` 自动执行

### 3. 查看结果

发布完成后，可在 Notion 中查看：

- 房源状态是否更新为 `posted`
- 发布时间是否已写入
- 后续互动和线索是否继续沉淀

## 图片建议

为了让小红书内容更像"真实经纪人笔记"而不是"硬广"，建议房源图片优先使用：

- 采光最好的客厅图
- 干净整洁的卧室图
- 小区外立面或园区图
- 周边配套图
- 户型图

推荐一套常用顺序：

1. 封面图：最吸引点击的一张
2. 客厅图：体现采光和空间感
3. 卧室图：体现居住舒适度
4. 厨卫图：体现真实维护情况
5. 小区或周边图：体现生活配套
6. 户型图：增强决策信息密度

当前也支持没有素材时先自动生成封面图，用于快速试跑流程。

## 常见问题

**Q：这套系统会不会被小红书封号？**

系统内置反垃圾规则：每天最多发 2 条、两条间隔 60 秒以上、禁止硬广话术。内容风格模拟真实经纪人笔记，不是批量营销号。但任何自动化工具都有平台风险，建议用非核心账号先跑通流程。

**Q：我不会用 Notion 怎么办？**

Notion 在这个系统里只是一个"录入表格"的角色。你只需要打开数据库，像填表一样填入房源信息就行。而且现在有了群聊监控，很多房源可以自动入库，连手动填表都省了。

**Q：企业微信需要额外付费吗？**

企业微信注册和使用完全免费。你只需要在 work.weixin.qq.com 注册，创建智能体和自建应用，5 分钟完成。团队成员用个人微信加入企业即可。

**Q：群聊监控会不会漏掉消息？**

系统通过 Websocket 实时连接企业微信，消息到达即处理。只要服务器在线，不会漏掉消息。每条识别出的房源都会查重后入库。

**Q：没有房源照片能发吗？**

可以。没有图片时系统会用 ImageMagick 自动生成一张渐变色文字封面，包含标题、价格、区域信息。视觉效果不如实拍照片好，但足够用来跑通流程和保持更新频率。

**Q：可以多人共用一个系统吗？**

可以。多人共用同一个 Notion 房源数据库录入，系统统一从数据库读取并发布。当前不支持按人分配不同小红书账号，这个在下一步增强计划中。

**Q：除了小红书，还能发到其他平台吗？**

当前只支持小红书。抖音已在技能中预留为 TODO，后续可扩展。
